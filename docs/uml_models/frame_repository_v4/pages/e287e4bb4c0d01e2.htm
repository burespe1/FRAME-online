Low Level Function
5.15.1.6 Detect Road Surface State in Host Vehicle vicinity

Overview

This Function shall be capable of providing the following facilities:

(1) The ability to take responsibility for processing information from sensors that detect the state of the road surface near to the Host Vehicle, i.e. the Vehicle in which it is operating.
(2) The ability to process the sensor data to determine if road surface is good, or if not what is the problem (e.g. slippery, broken) and why (e.g. oil spill, ice, mud, worn).
(3) Once processing is complete, the ability to send the data in digital form to the Determine and store local Host Vehicle Conditions Function.

Functional Requirements

(a) continuously monitor the flow of input data
(b) analyse the data in (a) to determine the state of the road surface immediately around the host vehicle
(c) where the road surface state is found to be less than good, determine the reason, e.g. ice, rain, oil, mud.
(d) if necessary convert the results from (b) and (c) into digital data
(e) whenever the state of the road surface changes and/or at other intervals, send the results of (d) to the Determine and store local Host Vehicle conditions function in the local vehicle road surface conditions data flow.
