Low Level Function
3.1.4.7 Provide Operator interface for Car Park Management

Overview

This Function shall be capable of providing the following facilities:

(1) A HMI through which the Parking Operator can manage one or a number of car parks.
(2) The HMI shall enable the Parking Operator to set up data about car parks that is loaded into the store of Car Park Data, to provide output of the current static data and to provide output of the current car park data (occupancy and/or number of spaces).
(3) The HMI shall also enable the Parking Operator to override the current status so that for example, a car park can be set to "CLOSED" regardless of how many of its spaces are not being used.

Functional Requirements

(a) when any of the input data flows are received from the Operator, process them and send the appropriate output data flows to either obtain the requested data or take the requested action
(b) when any of the data flows containing the responses to the output data flows in (a) are received, output the data they contain using the Operator output data flows.
