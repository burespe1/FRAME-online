Low Level Function
8.3.2.3 Monitor Cargo

Overview

This Function shall be capable of providing the following facilities:

(1) The ability to manage all data and processes that are concerned with the Cargo during the trip that it makes.
(2) The ability to provide data about the current state of the Cargo in response to requests received from other functionality.
(3) The ability to provide raw operational and data about the Cargo when an incident occurs.

Functional Requirements

(a) collect data (description, status, etc.) from the Cargo and store it in the store of On-board Data
(b) receive requests from the Fleet Manager and process them
(c) if and when an incident occurs, send an event to the Driver and/or the Fleet Manager and trigger associated task or transport order process
(d) give the Driver all functionality needed to access to cargo data in the store of On-board Data
(e) give any required information about the Cargo to other in-vehicle system non specific to FFM (emergency services, payment services, etc.)
(f) send the Cargo status to the Consignor/Consignee.
