Low Level Function
4.1.15 Provide PT Vehicle Driver Interface

Overview

This Function shall be capable of providing the following facilities:

(1) Support for both data and voice dialog between the PT Vehicle and Driver and the Collect PT Vehicle Data functionality through a suitable HMI for use in situations that are not included in updates provided by regularly transmitted data, e.g. emergencies, special messages to drivers, etc.
(2) The output of other PT Vehicle data and alarm information to the PT Driver through a suitable HMI.
(3) The ability of the PT Driver to use a suitable HMI to request information about the service (schedule and fares) on which the PT Vehicle is to be used and for that information to be displayed.

Functional Requirements

(a) continuously monitor for the arrival of input data flows from PT vehicle drivers;
(b) send messages from the PT vehicle driver to the Collect PT Vehicle Data function in the message from driver data flows
(c) continuously monitor for the arrival of the message to driver data flow
(g) when the data flow in (f) arrives immediately output its contents to the PT vehicle driver
(h) continuously monitor for the receipt of the assign vehicle to service data flow from the PT driver
(i) when the data flow in (h) arrives, put the identity of the service that it contains into the request schedule for vehicle and request fares for vehicle data flows and send them to the Plan New Service Schedule and Manage Fare Schemes Data Store functions
(j) as a result of (i) continuously monitor for the receipt of the requested service for vehicle and requested fares for vehicle data flows
(k) when the data flows in (j) arrive, store the data locally within the function.
