This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to receive and process a request from an entity that wants to become new NAP data provider.<br/>(2) The ability to check the new NAP data provider in (1) above to include identification, verification that the requesting entity exists and is believed to be capable of providing road related static and/or dynamic data based on information included in the request.<br/>(3) The ability to load information about the entity submitting the request in (1) above into the NAP Data Providers data store, if the identification, verification and acceptance in (2) above is successful.<br/>(4) The ability to provide authorisation credentials to the requesting entity.<br/>(5) The ability to send a rejection notice (including the reason for rejection) to the entity that submitted the request in (1) above if the identification, verification and acceptance in (2) and (3) above was not successful.<br/>(6) The ability to check the authorisation of a data provider which sends its credentials to the Function Provide Interface for Data Providers and to response to this Function if the authorisation check has been successful.<br/>(7) The ability to process the information provided in an unsuccessful request for the automatic update of NAP data so that a suitable response can be sent to the entity that made the request and future unsuccessful update requests monitored.<br/>
